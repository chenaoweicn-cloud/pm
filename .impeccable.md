## Design Context

### Users
- Primary user is a single PM / BA working on a Mac desktop throughout the day.
- They are managing 6-10 active projects and 100-300 active tasks across delivery, presales, operations, and follow-up work.
- The product acts as an external memory system: the user needs to answer "what projects am I holding?" quickly and later reconstruct weekly progress for manual reports.

### Brand Personality
- Calm
- Editorial
- Capable

The interface should feel composed and trustworthy, with a memorable point of view, but never ornamental or noisy. It should support focused office work rather than feel like a consumer app or a generic admin console.

### Aesthetic Direction
- Light theme, designed for daytime desktop use.
- Distinctive but restrained: memorable through composition, typography, and hierarchy rather than decoration.
- Anti-reference: common SaaS admin dashboards, dense BI panels, template-like card grids, generic "task manager" UIs.
- Visual metaphor: a curated working ledger or project folio laid open on a desk.

### Design Principles
- Show only the next useful layer of information; do not expose the whole system state at once.
- Keep active projects visible, but let one primary focus area dominate the page.
- Create memory through editorial rhythm, not through excessive color, chrome, or widgets.
- Make task language actionable and scannable in under two seconds.
- Prefer structured calm over dashboard busyness.
